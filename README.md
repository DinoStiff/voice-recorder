# 🚕 Taipei AI Tour Guide (AI 智慧導覽決策系統)

> 🏆 **這是一個專為黑客松打造的「高智能後端架構」，旨在透過即時環境上下文 (Context) 極大化 AI 推論的精準度！**

## 💡 我們解決了什麼痛點？ (The Problem)
在常見的 AI 行程規劃服務中，大語言模型 (LLM) 經常因為欠缺「現實世界的物理限制」而給出糟糕的體驗，例如：**推薦旅客在半夜三點去博物館，或是狂風暴雨時推薦去爬象山。**
如果把全台北幾百個景點的營業時間與型態一次丟給 AI 判斷，不僅容易產生幻覺 (Hallucination)，還會大幅消耗昂貴的 Token 成本。

## 🎯 我們的解法：極致的前置情境防火牆 (Our Solution)
我們打造了一套具備**活體更新**與**環境感知**功能的「情境防火牆與輿情探勘系統」。它攔截在使用者與大型語言模型之間，預先將最優質且「此時絕對可行」的資料萃取出來給前端及 AI 隊友。

---

## 🔥 三大系統亮點 (Key Highlights)

### 1. 🛡️ 智慧情境防火牆 (Context-Aware Firewall)
我們開發了專屬的 `/api/fetch_context` API。
當前端送入使用者的「現在時間 (Time)」與「外頭天氣 (Weather)」這兩個維度時，防火牆會：
- **時間排雷**：精準計算橫跨午夜的複雜營業時間，當下已經打烊的場所會被強制剔除。
- **天氣排雷**：遇到「雨天」等惡劣天候時，會直接遮蔽全無室內遮蔽的 `🌲 自然戶外` 景點。
經過這層防火牆過濾後，傳遞給 AI 推論大腦的陣列將達致 **100% 可用性**，徹底消滅 AI 胡亂推薦的幻覺。

### 2. 🌐 社群探勘與擴容引擎 (Dynamic POI Scraper)
系統打破靜態資料的侷限，內建了一支能持續將 **100+** 個北台灣重點觀光地段進行渲染的 `social_scraper.py`：
- **動態估值模型**：系統會自動依據景點標籤推算出合理的「每人預估消費 (Average Cost)」與「客製化交通方案」。
- **前端無縫接軌**：針對所有資料附上 9 大具代表性的 EMOJI 分類（如 `🍜 在地美食`, `🛍️ 商圈購物` 等），完全迎合現代扁平化且強視覺的 UI 介面。

### 3. ⚙️ 不死鳥背景自體更新生命週期 (Self-Updating Architecture)
伺服器底層導入了 FastAPI 非同步的 `Startup Event` 與 `asyncio`。
系統只要開啟，便會在底層默默開啟每 30 分鐘自動對外重啟爬蟲更新狀態的長駐 Loop。這意味著在黑客松或落地展示時，它完全不需要人工介入就能永遠吐出具備最新時間感知的內容。

## 🛠️ 如何啟動與 Demo (Quick Start)

若評審或開發者希望在本地驗證此模組的威力，只需兩行指令即可建置完成。

1. **安裝依賴模組**
```bash
pip install -r requirements.txt
```

2. **掛載智慧伺服器** (自帶背景爬蟲心跳偵測)
```bash
uvicorn api.index:app --reload
```

> **API 測試端點**: 發送 POST 請求至 `http://localhost:8000/api/fetch_context` ，並帶入 JSON `{"time": "23:30", "weather": "雨"}`，即可親眼見證無效戶外與日行景點遭到全數攔截的極簡回傳結果！
